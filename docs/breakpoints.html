Layout

Breakpoints

Los breakpoints son los puntos de quiebre donde el layout se adapta al tamaño del viewport. PersephoneCSS es Mobile First: los estilos base aplican en móvil y se sobreescriben hacia arriba.

¿Cómo funcionan?

PersephoneCSS define los breakpoints como variables SCSS en _variables.scss y los consume a través del mixin responsive() en _mixins.scss. Todos los media queries usan min-width, siguiendo el paradigma Mobile First: se escribe primero para móvil y luego se amplía para pantallas más grandes.

PersephoneCSS define solo dos breakpoints explícitos: md (768px) y lg (1024px). Los estilos sin breakpoint aplican a todos los tamaños, incluido extra small.

Mixin responsive()

Todos los media queries del framework se escriben a través del mixin responsive() definido en _mixins.scss. Acepta md o lg como argumento y genera el @media correspondiente.

_mixins.scss — Definición 
@mixin responsive($medida) {
  @if $medida == md {
    @media (min-width: $breakpoint-md) {
      @content;
    }
  } @else if $medida == lg {
    @media (min-width: $breakpoint-lg) {
      @content;
    }
  }
}
SCSS — Uso del mixin en un componente 
// Mobile First: base para móvil, se amplía hacia arriba
.col-6 {
  width: 100%;                    // xs — apilado
 
  @include responsive(md) {
    width: 50%;                   // md — dos columnas
  }
 
  @include responsive(lg) {
    width: 33.33%;                // lg — tres columnas
  }
}

Uso en el framework

El mixin responsive() se usa en múltiples parciales del framework. La tabla muestra qué cambia en cada componente por breakpoint.

Componente Parcial Cambio en md (768px) Cambio en lg (1024px)
Columnas grid _layout.scss Activa .col-md-{n} Activa .col-lg-{n}
Container _layout.scss max-width: 720px max-width: 1200px
Navbar _navbar.scss Nav-menu horizontal, oculta toggle Sin cambios adicionales
Sidebar _sidebar.scss Fijo y visible, desaparece toggle-btn Sin cambios adicionales
Footer _footer.scss Links en fila horizontal Sin cambios adicionales

Agregar un breakpoint personalizado

Para extender el framework con un nuevo breakpoint, solo se necesita agregar la variable en _variables.scss y el caso correspondiente en el mixin responsive().

Paso 1 — _variables.scss 
$breakpoint-md: 768px;
$breakpoint-lg: 1024px;
$breakpoint-xl: 1280px;  // ← nuevo breakpoint
Paso 2 — _mixins.scss 
@mixin responsive($medida) {
  @if $medida == md {
    @media (min-width: $breakpoint-md) { @content; }
  } @else if $medida == lg {
    @media (min-width: $breakpoint-lg) { @content; }
  } @else if $medida == xl {
    @media (min-width: $breakpoint-xl) { @content; }  // ← nuevo caso
  }
}
Paso 3 — Uso en _layout.scss 
@include responsive(xl) {
  @for $i from 1 through 12 {
    .col-xl-#{$i} {
      $width: (100% * $i / 12);
      flex: 0 0 $width;
      max-width: $width;
    }
  }
}
Al centralizar los valores en variables y el mixin en un solo lugar, extender el sistema de breakpoints no requiere tocar ningún otro parcial del framework.
