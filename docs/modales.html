Componentes

Modales

Ventanas emergentes con overlay. Tres tipos: base, alerta y formulario. Se activan con data-modal-target y se cierran con data-modal-close o haciendo click en el overlay.

¿Cómo funciona?

El modal permanece oculto por defecto. El JavaScript escucha clicks en elementos con data-modal-target="{id}" y añade .is-open al modal correspondiente para mostrarlo. Cualquier elemento con el atributo data-modal-close lo cierra, al igual que hacer click fuera del modal sobre el overlay.

_modal.scss — Tipos disponibles 
.modal       { @include modal-base; }   // base, max-width 500px
.modal-alert { @include modal-base; }   // alerta, max-width 400px centrado
.modal-form  { @include modal-base; }   // formulario, max-width 600px
 
// Variante de color (solo para .modal)
.modal.modal-primary { @include modal-color($primary); }
El id del modal debe coincidir exactamente con el valor de data-modal-target del botón que lo abre.

Elementos del modal

Estructura interna compartida por los tres tipos de modal.

Clase / Atributo Elemento Descripción
.modal / .modal-alert / .modal-form <div> Contenedor principal con overlay. Oculto por defecto
.is-open modificador JS la añade para mostrar el modal
.modal-content <div> Caja centrada con fondo y sombra
.modal-header <div> Cabecera con título y botón de cierre
.modal-body <div> Área de contenido principal
.modal-footer <div> Pie con botones de acción
data-modal-target="{id}" atributo En el disparador: indica qué modal abrir
data-modal-close atributo En cualquier elemento: cierra el modal activo

Modal base — .modal

El modal estándar para mostrar información o confirmar acciones. Ancho máximo de 500px.

Título del modal

Este es el contenido del modal base. Puedes colocar cualquier elemento HTML dentro del .modal-body: texto, imágenes, listas, cards, etc.

Incluso alertas del framework funcionan dentro del modal.

Modal con color primary

La clase .modal-primary añadida junto a .modal aplica el color primario al .modal-header.

HTML — Modal base 
<!-- Botón disparador -->
<button class="btn btn-primary btn-md" data-modal-target="miModal">
  Abrir modal
</button>
 
<!-- Modal -->
<div class="modal" id="miModal">
  <div class="modal-content">
    <div class="modal-header">
      <h3>Título</h3>
      <button data-modal-close>×</button>
    </div>
    <div class="modal-body">
      <p>Contenido del modal.</p>
    </div>
    <div class="modal-footer">
      <button class="btn btn-sm btn-outline-secondary" data-modal-close>Cancelar</button>
      <button class="btn btn-sm btn-primary">Confirmar</button>
    </div>
  </div>
</div>
HTML — Variante .modal-primary 
<div class="modal modal-primary" id="miModal">
  ...
</div>

Modal alerta — .modal-alert

Para confirmaciones de acciones destructivas. Contenido y botones centrados. Ancho máximo 400px.

¿Eliminar registro?

Esta acción no se puede deshacer. El registro será eliminado permanentemente.

¿Continuar?

Esta acción modificará datos existentes. ¿Deseas continuar?

HTML — Modal alerta 
<!-- Botón disparador -->
<button class="btn btn-danger btn-md" data-modal-target="confirmarEliminar">
  Eliminar
</button>
 
<!-- Modal alerta -->
<div class="modal-alert" id="confirmarEliminar">
  <div class="modal-content">
    <div class="modal-header">
      <h3>¿Eliminar registro?</h3>
      <button data-modal-close>×</button>
    </div>
    <div class="modal-body">
      <p>Esta acción no se puede deshacer.</p>
    </div>
    <div class="modal-footer">
      <button class="btn btn-sm btn-outline-secondary" data-modal-close>Cancelar</button>
      <button class="btn btn-sm btn-danger">Sí, eliminar</button>
    </div>
  </div>
</div>

Modal formulario — .modal-form

Optimizado para formularios. Ancho máximo 600px, mayor padding en el body y footer con justify-content: space-between.

Nuevo registro

Iniciar sesión

HTML — Modal formulario 
<!-- Botón disparador -->
<button class="btn btn-secondary btn-md" data-modal-target="modalRegistro">
  Nuevo registro
</button>
 
<!-- Modal formulario -->
<div class="modal-form" id="modalRegistro">
  <div class="modal-content">
    <div class="modal-header">
      <h3>Nuevo registro</h3>
      <button data-modal-close>×</button>
    </div>
    <div class="modal-body">
      <div class="form-group">
        <label class="label">Nombre</label>
        <input type="text" class="input" placeholder="Tu nombre"/>
      </div>
      <div class="form-group">
        <label class="label">Correo</label>
        <input type="email" class="input" placeholder="correo@ejemplo.com"/>
      </div>
    </div>
    <div class="modal-footer">
      <button class="btn btn-sm btn-outline-secondary" data-modal-close>Cancelar</button>
      <button class="btn btn-sm btn-secondary">Guardar</button>
    </div>
  </div>
</div>

Resumen de tipos

Diferencias entre los tres tipos de modal disponibles en el framework.

Tipo Clase Max-width Alineación footer Uso ideal
base .modal 500px flex-end Información general, confirmaciones
color .modal .modal-primary 500px flex-end Modal base con header de color primary
alerta .modal-alert 400px center Acciones destructivas, confirmaciones críticas
formulario .modal-form 600px space-between Registro, edición, login, cualquier formulario
Coloca los modales justo antes del cierre de </body> para evitar conflictos de z-index con otros componentes.
